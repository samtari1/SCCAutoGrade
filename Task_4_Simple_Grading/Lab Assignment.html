Lab Assignment: Creating Views

Topic: Views – Creation, Benefits, and Security

Important: Before starting this lab, please first create the university database by running the setup SQL below: 
-- 1. Create database
CREATE DATABASE IF NOT EXISTS university;
USE university;

-- 2. Create tables

-- Students table
CREATE TABLE students (
    student_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    major VARCHAR(50),
    gpa DECIMAL(3,2)
);

-- Courses table
CREATE TABLE courses (
    course_id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(100),
    credits INT
);

-- Professors table
CREATE TABLE professors (
    professor_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    department VARCHAR(50)
);

-- Enrollments table (relationship between students and courses)
CREATE TABLE enrollments (
    enrollment_id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT,
    course_id INT,
    grade DECIMAL(5,2),
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (course_id) REFERENCES courses(course_id)
);

Objectives

  • Understand the benefits of using views in MySQL.
  • Create views from single or multiple tables.
  • Use column aliases, aggregate functions, and the LIMIT clause in views.
  • Apply views to enhance database security by restricting access to specific columns or rows.
  • Query data from views and combine views with other SQL operations.

Lab Instructions

Part 1: Simple View

Create a view that shows only student_id and first_name from the students table. Query the view to display all records.

Part 2: View with JOIN

Create a view that shows student_name, course_name, and grade by joining students and enrollments tables (and optionally courses if needed). Query the view to see all records.

Part 3: Aggregate View

Create a view showing each student's average grade across all courses. Use aggregate functions and GROUP BY. Query the view and sort by average grade descending.

Part 4: Security with Views

Create a view that shows student names and majors but hides grades. Grant SELECT privileges on this view to a user report_user. Verify that report_user cannot access the hidden columns.

Part 5: Advanced View

Create a view showing the top 5 students in "Computer Science" by average grade. Include student name, major, and average grade. Query the view to verify the results.

Part 6: Combined View Usage

Create a view that contains the GPA column and write a Select statement to query on the view to filter students with GPA greater than 3.5 and order them by last name.

Part 7: Short-Answer Questions

  1. What are the main benefits of using views in MySQL?
  2. How do views improve database security?
  3. How does a view help hide table structure changes from users?
  4. Why might you use an aggregate function in a view?
  5. Explain the difference between querying a table directly and querying through a view.
